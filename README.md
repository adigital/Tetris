# 🤖 Проект: Автоматический игрок в Тетрис (на базе Android + ESP32)

## 📌 Описание проекта

Цель проекта — создать физический «игровой автомат», который самостоятельно играет в классический карманный тетрис (brick game), используя:

- 📱 смартфон на Android (как «глаза» и «мозг»)
- 🎥 камеру для распознавания игрового поля
- 🔌 микроконтроллер для управления
- ⚙️ сервоприводы для физического нажатия кнопок

👉 Важно: устройство **не разбирается**, управление осуществляется **только через физическое нажатие кнопок**.

---

## 🎯 Цели проекта

### Основные цели:
- Распознавание игрового состояния через камеру смартфона
- Анализ текущей ситуации в игре (логика тетриса)
- Принятие решения (куда двигать фигуру)
- Передача команд на микроконтроллер
- Физическое нажатие кнопок

---

### Дополнительные цели:
- Минимизация стоимости и сложности сборки
- Полностью автономная работа
- Возможность расширения (другие игры, алгоритмы)

---

## 🧠 Архитектура системы

[Android смартфон]
↓ (камера + CV)
[Алгоритм / логика]
↓ (Bluetooth)
[ESP32]
↓ (GPIO)
[Сервоприводы]
↓
[Кнопки тетриса]

## 🔧 Текущее оборудование

### 📱 Смартфон
- Android устройство
- Используется для:
  - камеры
  - обработки изображения
  - логики игры
  - передачи команд

---

### 🔌 Микроконтроллер
- ESP32 DevKit
- Возможности:
  - Bluetooth (BLE)
  - GPIO для управления сервами
  - программирование через USB

---

### ⚙️ Сервоприводы
- Тип: SG90 (micro servo)
- Количество: 4
- Назначение:
  - физическое нажатие кнопок (влево, вправо, поворот, вниз)

---

### 🔲 Макетная плата (breadboard)
- 400 точек
- Используется для:
  - коммутации питания
  - подключения сервоприводов
  - подключения ESP32

---

### 🔌 Провода
- Dupont (male-male)
- Используются для:
  - соединения сервоприводов с breadboard
  - подключения ESP32

---

### 🔋 Питание
- Внешний источник 5V (3–5A)
- Используется для питания сервоприводов

---

### 🧱 Механическая конструкция
- Основание (доска/пластик)
- Держатель для тетриса (съёмный)
- Крепления для сервоприводов
- Держатель смартфона

---

## 🔌 Схема подключения (логика)

### Один сервопривод:

| Провод | Подключение |
|--------|-------------|
| Красный | +5V (внешнее питание) |
| Коричневый | GND |
| Оранжевый | GPIO ESP32 |

---

### Важно:
- Все GND должны быть объединены
- Питание сервоприводов НЕ от ESP32
- ESP32 используется только для сигналов

## 💻 Необходимое ПО

### 🧑‍💻 Для ESP32
- Arduino IDE  
или  
- PlatformIO (VS Code)

#### Библиотеки:
- Servo / ESP32Servo
- Bluetooth (BLE)

---

### 📱 Для Android

#### Основные компоненты:
- Kotlin (рекомендуется)
- Camera API
- OpenCV (или аналог)

---

### 🔍 Модули приложения:

1. Захват изображения (камера)
2. Обработка изображения:
   - бинаризация
   - выделение сетки
3. Распознавание фигур
4. Алгоритм игры:
   - оценка позиции
   - выбор действия
5. Передача команд (BLE)

---

## 🔄 Поток данных

1. Камера снимает экран тетриса  
2. Алгоритм распознаёт состояние  
3. Вычисляется оптимальное действие  
4. Команда отправляется на ESP32  
5. ESP32 двигает сервопривод  
6. Сервопривод нажимает кнопку  

---

## ⚠️ Ограничения и нюансы

- Требуется стабильное положение камеры
- Задержки между нажатиями (~100 мс)
- Точная механическая калибровка
- Ограничение по току (сервоприводы)
